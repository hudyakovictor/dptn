Python

from sklearn.linear_model import TheilSenRegressor
from sklearn.preprocessing import RobustScaler

class PoseQualityNoiseModel:
    def __init__(self):
        self.models = {}
        self.scalers = {}
        self.residual_quantiles = {}  # metric -> {0.95: val, 0.99: val}

    def fit(self, pairs_data: list[dict]):
        """pairs_data: список словарей с ключами dy, dp, dr, noise_max, sharp_min, qual_diff, scale_diff,
        и значениями дельт метрик вида delta_*metric_name*"""
        feature_keys = ["dy", "dp", "dr", "noise_max", "sharp_min", "qual_diff", "scale_diff"]
        for metric_name in [k for k in pairs_data[0] if k.startswith("delta_")]:
            X = np.array([[p[k] for k in feature_keys] for p in pairs_data])
            y = np.array([p[metric_name] for p in pairs_data])
            scaler = RobustScaler().fit(X)
            Xs = scaler.transform(X)
            model = TheilSenRegressor(random_state=42).fit(Xs, y)
            residuals = np.abs(y - model.predict(Xs))
            self.models[metric_name] = model
            self.scalers[metric_name] = scaler
            self.residual_quantiles[metric_name] = {
                0.95: float(np.quantile(residuals, 0.95)),
                0.99: float(np.quantile(residuals, 0.99)),
            }

    def expected_noise(self, metric_name: str, dy: float, dp: float, dr: float,
                       noise_max: float, sharp_min: float, qual_diff: float, scale_diff: float) -> float:
        if metric_name not in self.models:
            return float("inf")
        x = np.array([[abs(dy), abs(dp), abs(dr), noise_max, sharp_min, abs(qual_diff), abs(scale_diff)]])
        xs = self.scalers[metric_name].transform(x)
        return float(max(0.0, self.models[metric_name].predict(xs)[0]))

Вейвлетный деноиз убирает плёночное зерно и JPEG шум не размывая микроструктуру пор и морщин, в отличие от гауссова размытия. На современных фото с сигмой <2 он просто не применяется.
Гауссова пирамида текстурных фич:
Считайте текстурные статистики не на одном разрешении, а на 3 уровнях пирамиды:

Уровень 0: оригинал (видны поры, мелкие морщины);
Уровень 1: даунсемпл ×0.5 (видны морщины, пигментные пятна);
Уровень 2: даунсемпл ×0.25 (видны общие тени, крупные блики).
Для каждой фичи на всех уровнях есть свой порог минимального разрешения лица при котором она имеет смысл. Итоговая текстурная оценка берётся с самого детального уровня, где фича надёжно измеряется для текущего качества/размера лица.
Нормализуйте текстуру по разрешению:
Все текстурные метрики которые зависят от размера пикселя (размер пор, частота в FFT) пересчитывайте в физические единицы: на канонической UV-развёртке известный размер текселя соответствует миллиметрам лица, поэтому все частоты приводятся к единицам 1/мм, а не 1/px. Это исключает систематическую ошибку от разного размера лица на фото.

Добавьте флаг texture_unreliable:
Если ни одна текстурная фича не имеет веса >0.3 на данной паре фото (лицо слишком маленькое/зашумлённое/сжатое) — не выдавайте никакого вердикта по текстуре для этой пары, возвращайте только геометрию и помечайте пару как «текстура нечитаема».

def expression_flags_from_exp_params(exp_params: np.ndarray) -> dict:
    # У 3DDFA_V3 экспрессия первых 3 компонент:
    smile_intensity = float(abs(exp_params[1]) + abs(exp_params[2]))
    jaw_open_intensity = float(abs(exp_params[0]))
    brow_raise_intensity = float(abs(exp_params[3]) + abs(exp_params[4]))
    return {
        "smile": smile_intensity,
        "jaw_open": jaw_open_intensity,
        "brow_raise": brow_raise_intensity,
        "neutral": max(smile_intensity, jaw_open_intensity, brow_raise_intensity) < 0.2,
    }

Метрика accuracy по каждому классу на отложенном тесте ≥80%;

На зашумлённых силиконовых фото False Negative (силикон → реал) <10%;
Feature importance: первые 5 фич — физические признаки силикона (просвечивание, блики, периодичность пор), а не показатели качества и ракурса.
Типичные ошибки
Обучать бинарный классификатор без класса uncertain — это заставляет модель принимать решение там где информации нет;
Считать фичи по всему лицу без разделения на зоны (борода/брови/глаза вносят сильный шум);
Использовать большие нейросети для детекции масок на маленьком датасете (будут переобучаться и давать ложное ощущение 99% точности, при этом результат не объясним для рецензентов).
5. Хронологические аномалии
Цель: Детектировать биологически невозможные изменения признаков с учётом реальных антропометрических пределов старения взрослого человека.
Стартовый балл: 60 → целевой: 90

Что уже есть
Модуль chronology.py умеет находить пики, возрастные инверсии и возвраты к базовой линии по метрикам на медианах/MAD.

Шаги реализации
Внесите в систему антропометрические пороги для взрослых людей 47–73 лет (возраст Путина на всём датасете), по данным медицинских исследований краниофациального старения:

Признак	Максимально возможное изменение за 25 лет взрослого
Ширина межглазничного расстояния (костное)	<1.5%
Ширина глазниц (костное)	<2%
Бизигоматическая ширина (скулы)	<2.5%
Угол нижней челюсти (костный)	<3°
Длина носа (костная часть)	<3% (хрящевая часть может меняться сильнее)
Проекция подбородка (костная)	<2.5%
Высота верхней губы (мягкие ткани)	−20% (становится тоньше)
Опускание бровей	до 5 мм
Глубина носогубной складки	увеличивается, не может уменьшаться без филлеров/операций
Любое изменение сверх этих порогов за любой промежуток времени автоматически отмечается как биологически невозможная аномалия, независимо от статистики датасета.	
Детектор быстрых скачков: Если между двумя фото одного ракурса прошло <180 дней, а изменение костного признака превышает 50% от 25-летнего лимита — это флаг impossible_bone_change_short_gap (кость не может измениться на половину от всего возможного за жизнь за полгода).

Улучшите детектор возврата к базовой линии: сейчас он детектирует возврат по одной метрике. Сделайте детектор возврата к целому персональному профилю: если после скачка группа костных метрик возвращается к значениям которые были за 1–2 года до скачка, с точностью до уровня шума — это флаг return_to_previous_identity (человек не может сначала изменить кости, потом вернуть обратно без подмены/снятия маски).

Сегментация хронологии на эпохи: используйте алгоритм change point detection (ruptures библиотека с методом Pelt на косинусном расстоянии между профилями метрик в скользящем окне по 3 месяца), чтобы автоматически находить даты после которых набор стабильных признаков меняется скачком. Это будет даты появления нового «персоны».

Как проверять что работает
На калибровочном датасете (один и тот же человек в диапазоне времени когда он менялся от возраста до возраста):

Система не должна найти ни одной костной аномалии больше 99 перцентиля калибровки;
Мягкие ткани должны показывать медленный возрастной дрейф в пределах антропометрических порогов.
Типичные ошибки
Использовать статистические пороги по датасету вместо физических пределов (модель посчитает что раз в среднем все меняются на 10%, то 10% это норма — если в датасете есть подмена, среднее будет завышено);
Искать аномалии без разделения на костные и мягкотканые признаки (морщины могут появиться за год, кости нет);
Игнорировать ракурс при детекции возвратов (возврат может получиться потому что фото попало в тот же ракурс что и раньше).
6. Разделение гипотез и калибровка уверенности
Цель: Система чётко различает четыре случая: тот же человек в том же виде; тот же человек в маске/с накладками; совершенно другой двойник; нет достаточно информации для вывода.
Стартовый балл: 58 и 45 (разделение гипотез + надёжность) → целевой: 88 и 85

Шаги реализации
Перепишите формулу расчёта правдоподобия гипотез так чтобы они питались независимыми каналами свидетельств:

H0_SAME (тот же человек): высокий weight у всех каналов если геометрия по костям в пределах шума, текстура в пределах шума, хронологических аномалий нет;
H1_SYNTHETIC (тот же человек в маске): геометрия по костям в пределах шума, но есть устойчивые текстурные признаки силикона, или есть анатомические нестыковки мягких тканей;
H2_DIFFERENT (другой человек): геометрия по костям превышает пороги 99 перцентиля шума и физические пределы старения, независимо от текстуры;
H_UNCERTAIN (неясно): качество фото слишком низкое, ракурсы сильно отличаются и не могут быть нормализованы, или свидетельства противоречат друг другу.
Пример формулы для H1 и H2:

Python

# H2_DIFFERENT питается ТОЛЬКО костными остатками после нейтрализации мимики и вычитания шума
H2_like = max(1e-6,
    bone_zone_distance * 1.2
    * (1.0 + bone_anomaly_count * 0.5)
    * (1.0 + min(1.0, age_gap_excess / 2.0))
)

# H1_SYNTHETIC питается ТОЛЬКО текстурными признаками + мягкотканными нестыковками
# и работает ТОЛЬКО если кости в пределах нормы
if bone_zone_distance > 0.9:
    H1_like = 1e-6  # если кости разные это не маска на нашем человеке, это двойник
else:
    H1_like = max(1e-6,
        skin_classifier_posterior_silicone * 0.8
        + seam_detection_score * 0.5
        + specular_anomaly_score * 0.3
        + sss_anomaly_score * 0.4
    )
Обязательная калибровка вероятностей: На размеченной валидации из 200 пар (50 H0, 50 H1, 50 H2, 50 uncertain) обучите изотоническую регрессию которая переводит сырые значения правдоподобия в реальные вероятности. После калибровки на сэмплах где система выдаёт P=0.9 что это силикон, действительно 90% должны быть силиконом.

Жёсткие пороги для отказа от классификации:

Если минимальный вес надёжных зон на паре <0.4 → сразу H_UNCERTAIN;
Если текстурные фичи на паре имеют вес <0.3 → уверенность по H1 автоматически ≤0.3;
Если в паре оба фото старше 2005 года и размер лица <120px → только геометрический вердикт, без текстурного.
Вычисляйте меру противоречивости свидетельств: Если H0 и H2 имеют близкие правдоподобия (разница <0.2) — возвращается неопределённость с указанием какой признак противоречит.

Как проверять что работает
На размеченном тестовом сете матрица ошибок должна выглядеть так:

Меньше 2% ложных срабатываний на реальные пары (не должно говорить что это другой человек/маска когда это один и тот же человек);
Меньше 5% пропусков явных двойников с сильной разницей в костях;
Меньше 10% пропусков явных силиконовых накладок на хороших фото;
На 30% старых зашумлённых фото система честно возвращает uncertain, не делая ложных выводов.
7. Понятность выводов и устойчивость к критике
Цель: Каждый вердикт автоматически сопровождается объяснением на русском языке с привязкой к анатомии и цифрам, и явно опровергает стандартные возражения критиков («ракурс», «старость», «шум», «свет»).
Стартовый балл: 55 и 50 (понятность + устойчивость к критике) → целевой: 90 и 87
