**Фикс:** Убрать pose из фичей классификатора. Добавить качественные инвариантные фичи (FFT, LBP-энтропия, albedo).

### 2.2. Pose-invariant текстурные метрики
Убрать yaw/pitch/roll из фичей классификатора. Добавить:

```python
# FFT-анализ частот (по TZ required)
def fft_skin_score(texture_map: np.ndarray, face_mask: np.ndarray) -> dict:
    """
    Силиконовые маски имеют аномальную периодичность в высоких частотах.
    Настоящая кожа — хаотичная, с пиками на частотах пор (0.3–0.8 мм).
    """
    gray = cv2.cvtColor(texture_map, cv2.COLOR_RGB2GRAY)
    gray = gray[face_mask > 0]

    # FFT по патчам
    patches = extract_patches(gray, size=64, stride=32)
    scores = []
    for patch in patches:
        f = np.fft.fft2(patch)
        power = np.abs(f)**2
        # Отношение высоких частот к низким
        h, w = patch.shape
        cy, cx = h//2, w//2
        yy, xx = np.ogrid[:h, :w]
        radius = np.sqrt((yy-cy)**2 + (xx-cx)**2)

        low = power[radius <= 4].sum()
        high = power[radius > 8].sum()
        scores.append(high / (low + 1e-6))

    return {
        "fft_highfreq_ratio_mean": np.mean(scores),
        "fft_highfreq_ratio_std": np.std(scores),
        "fft_peak_regularity": measure_peak_regularity(power),  # силикон = регулярные пики
    }

# LBP-сложность (по TZ required)
def lbp_complexity(texture_map: np.ndarray, face_mask: np.ndarray) -> dict:
    """
    Настоящая кожа: высокая энтропия LBP (поры, морщины, микрорельеф).
    Силикон: низкая энтропия, однородные паттерны.
    """
    from skimage.feature import local_binary_pattern
    gray = cv2.cvtColor(texture_map, cv2.COLOR_RGB2GRAY)
    gray = gray[face_mask > 0]

    lbp = local_binary_pattern(gray, P=8, R=1, method="uniform")
    hist, _ = np.histogram(lbp, bins=59, range=(0, 59), density=True)
    entropy = -np.sum(hist * np.log2(hist + 1e-10))

    # Multi-scale LBP
    lbp_r2 = local_binary_pattern(gray, P=16, R=2, method="uniform")
    hist_r2, _ = np.histogram(lbp_r2, bins=243, range=(0, 243), density=True)
    entropy_r2 = -np.sum(hist_r2 * np.log2(hist_r2 + 1e-10))

    return {
        "lbp_entropy_r1": entropy,
        "lbp_entropy_r2": entropy_r2,
        "lbp_complexity_ratio": entropy_r2 / (entropy + 1e-6),
    }

# Color albedo (по TZ required)
def albedo_viability(texture_map: np.ndarray, face_mask: np.ndarray) -> dict:
    """
    Жизнеспособность кожи: анализ сосудистого рисунка и гемоглобина.
    Силикон: плоский спектр, отсутствие пика в 540–580 нм (гемоглобин).
    """
    skin = texture_map[face_mask > 0]
    # Конвертация в LAB
    lab = cv2.cvtColor(texture_map, cv2.COLOR_RGB2LAB)
    l, a, b = lab[:,:,0], lab[:,:,1], lab[:,:,2]

    # Канал a* в LAB: зелёный-красный. Живая кожа имеет вариативность.
    a_skin = a[face_mask > 0]

    return {
        "albedo_a_mean": np.mean(a_skin),
        "albedo_a_std": np.std(a_skin),
        "albedo_viability_index": np.std(a_skin) / (np.mean(l[face_mask > 0]) + 1e-6),
```

## ЭТАП 3: КАЛИБРОВКА (S3) — РЕАЛЬНЫЙ BASELINE

### 3.1. Калибровочный шум ракурса
Для каждого bucket (например, `left_threequarter_light`):
1. Берём все твои калибровочные фото в этом bucket.
2. Для каждой пары вычисляем:
   - `pose_gap = sqrt((1.4*dyaw)^2 + dpitch^2 + (0.6*droll)^2)`
   - `geometry_distance` по костным метрикам
   - `texture_distance` по текстурным метрикам
3. Строим регрессию: `distance ~ pose_gap + quality`.
4. Получаем **ожидаемый шум** для данного pose_gap.

### 3.2. Применение к основному датасету
При сравнении двух фото из основного датасета:
```python
expected_noise = calibration_model.predict(pose_gap, quality_a, quality_b)
actual_distance = compute_distance(photo_a, photo_b)
excess_distance = max(0, actual_distance - expected_noise)
```
`excess_distance` — это и есть сигнал "разные лица", а не шум ракурса.

## ЭТАП 4: СРАВНЕНИЕ (S4) — POSE-ALIGNED PAIRS

### 4.1. ICP-выравнивание 3D-мешей
Перед сравнением двух мешей:
```python
def align_meshes(v_a: np.ndarray, v_b: np.ndarray) -> tuple[np.ndarray, np.ndarray]:
    """
    Procrustes alignment: выравниваем v_b к v_a по костным якорям.
    """
    # Используем только костные точки (индексы вершин)
    bone_indices = get_bone_vertex_indices()  # ~500 вершин на костных зонах

    va = v_a[bone_indices]
    vb = v_b[bone_indices]

    # Центрирование
    ca, cb = va.mean(axis=0), vb.mean(axis=0)
    va_c, vb_c = va - ca, vb - cb

    # SVD для нахождения оптимального поворота
    H = vb_c.T @ va_c
    U, S, Vt = np.linalg.svd(H)
    R = Vt.T @ U.T

    # Применяем к полному мешу b
    v_b_aligned = (v_b - cb) @ R + ca
    return v_a, v_b_aligned
```

### 4.2. Тепловая карта различий
```python
def heatmap_difference(v_a: np.ndarray, v_b: np.ndarray, triangles: np.ndarray) -> np.ndarray:
    """
    Возвращает per-vertex разницу в мм.
    """
    diff = np.linalg.norm(v_a - v_b, axis=1)  # (N,) — расстояние для каждой вершины

    # Нормализация по зонам
    for zone_name, zone_indices in ZONE_INDICES.items():
        zone_diff = diff[zone_indices]
        # Костные зоны: порог жёсткий (2 мм)
        # Мягкие ткани: порог мягкий (5 мм)
        threshold = 2.0 if zone_name in BONE_ZONES else 5.0
        diff[zone_indices] = np.clip(zone_diff / threshold, 0, 1)

    return diff
```

```python
def check_biological_impossibility(photo_a, photo_b, metrics_a, metrics_b):
    flags = []
    gap_days = abs((photo_a.date - photo_b.date).days)

    # 1. Костные метрики изменились слишком быстро
    bone_change = abs(metrics_a["bone_zygomatic_width"] - metrics_b["bone_zygomatic_width"])
    if bone_change > 2.0 and gap_days < 365:
        flags.append("IMPOSSIBLE_BONE_CHANGE:zygomatic")

    # 2. Инверсия асимметрии
    asym_a = metrics_a["bone_asymmetry_x"]
    asym_b = metrics_b["bone_asymmetry_x"]
    if (asym_a > 2.0 and asym_b < -2.0) or (asym_a < -2.0 and asym_b > 2.0):
        flags.append("IMPOSSIBLE_ASYMMETRY_INVERSION")

    # 3. "Возврат к baseline" — форма менялась, потом вернулась
    # (маркер маски: одна маска в 2015, другая в 2020, но кости "вернулись")
```

# ⚠️ ПРЕДУПРЕЖДЕНИЕ О ГИПОТЕЗЕ

Ты правильно отмечаешь, что "двойники" — не обязательно близнецы, а люди с похожей формой черепа + силикон + грим. Это означает:

1. **Костные метрики** будут показывать **частичное совпадение** (черепа похожи, но не идентичны).
2. **Текстура** — главный маркер: силиконовые накладки имеют другой микрорельеф, albedo, блики.
3. **Хронология** — ключевой доказательный ряд: если "один человек" демонстрирует 3 разных костных структуры в разные периоды — это не один человек.

Поэтому в отчёте нужно избегать категоричных формулировок вроде "это двойник". Правильно: **"данное фото демонстрирует костные/текстурные характеристики, несовместимые с установленным baseline периода X, с вероятностью подмены Y%"**.
